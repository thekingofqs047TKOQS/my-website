What You Will Learn In This Course

How to use Tailwind classes

We will explore the most important Tailwind CSS classes and use them frequently in practice. Tailwind offers many helpful utilities such as spacing, background colors, borders, shadows, and typography. By repeatedly typing and applying these classes in real examples, we reinforce learning through repetition, which becomes our strongest tool for mastering Tailwind.

How to make good-looking designs

There is a simple, repeatable process for designing with Tailwind that helps us create attractive and professional designs. By following a few easy-to-remember rules, we can consistently achieve good visual results. We also look at ideas and practical action steps that take a design from simply “good” to truly great.

Why Tailwind is different or better

At first, Tailwind may not seem appealing, and it can even feel like using inline styles. However, with continued use, many developers grow to appreciate how powerful and flexible it is. Over time, Tailwind can become a favorite tool, to the point where starting a project without it feels uncomfortable.

How to build this site

The website being referenced is built entirely with Tailwind CSS and does not use any custom CSS. All of the site’s code is available to reuse for your own landing pages or websites.

Learners can study the real project structure and benefit from seeing exactly how Tailwind is applied in a full site.

A fun and methodical design process

Design does not always come naturally, and many people do not see themselves as designers. A helpful approach is to follow a clear formula instead of guessing. A practical order to think about design is spacing first, then box properties, then typography, and finally adding a fun element that makes the design stand out. This structured process is emphasized throughout learning.

Creating common components

One of the first things we want to do when using Tailwind is to create components like cards, buttons, and forms. We’ll build these out and see how Tailwind compares to CSS frameworks like Bootstrap.

We’ll also build out navigation, hero, pricing, newsletter, and footer components in this course.

Shrinking bundle sizes

Tailwind comes with so many classes. It doesn’t make sense to send all of those to our users’ browsers. With PostCSS and PurgeCSS, we can shrink the amount of CSS we send to our users.

We’ll learn how to only send the classes that we actually use so that our bundle sizes can drop to even below 10kb!

Using Tailwind w/ React

Tailwind’s benefits really shine when you pair it with a JavaScript library/framework. Extracting classes into components is the recommended way to keep Tailwind DRY (don’t repeat yourself).

While we could use the @apply() or theme() functions, it forgoes some of the benefits of Tailwind. Namely we are once again creating a custom class and CSS.

How to customize Tailwind

Tailwind is easily configurable for our own projects using the tailwind.js config file. We’ll explore changing out fonts, colors, sizes, and more so that our Tailwind setups are custom and tailored to our projects.

Using Tailwind in VS Code

VS Code is my favorite editor and Tailwind works fantastically in it. There are some essential plugins and setup for using Tailwind in VS Code. We’ll go over each and use VS Code heavily in this course.

Extending Tailwind

Learning Tailwind is just the beginning. There are some great tools like Tailwind Typography, Tailwind UI, Tailwind Builder, and more. We’ll talk about some ways to extend Tailwind and ways we can go further.
